01

Launch an EC2 Instance

AWS Console → EC2 → Ubuntu 22.04

Start by launching a Linux server on AWS. I used Ubuntu 22.04 LTS on a t3.small instance — enough for personal use with a small number of users.

SettingValue
Instance Typet3.small (2 vCPU, 2GB RAM minimum)
AMIUbuntu Server 22.04 LTS
Storage50GB GP3 (MongoDB grows over time)
Key PairCreate or use existing .pem key

Security Group — open these ports:

PortProtocolSourcePurpose
22TCPYour IPSSH access
80TCP0.0.0.0/0HTTP (Certbot verification)
443TCP0.0.0.0/0HTTPS (your site)
Tip: Assign an Elastic IP to your instance. Without it, the public IP changes every time the server restarts — which would break your DNS record.
Why t3.small and not t3.micro?
LibreChat runs 5 containers simultaneously (app, MongoDB, Meilisearch, RAG API, vector DB). A t3.micro with 1GB RAM will run out of memory. t3.small at 2GB is the practical minimum.
02

Install Docker

SSH into server → install Docker Engine

SSH into your new EC2 instance and install Docker. LibreChat uses Docker Compose to manage all its services — no manual installation of MongoDB or any other dependency needed.

# Update package list
sudo apt update && sudo apt upgrade -y

# Install prerequisites
sudo apt install -y ca-certificates curl gnupg

# Add Docker's official GPG key
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
  | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# Add Docker repository
echo "deb [arch=$(dpkg --print-architecture) \
  signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" \
  | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Install Docker Engine + Compose plugin
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io \
  docker-buildx-plugin docker-compose-plugin

# Allow current user to run Docker without sudo
sudo usermod -aG docker $USER
newgrp docker

# Verify
docker --version
docker compose version
03

Install LibreChat

clone → configure → docker compose up

Clone the LibreChat repository and set up the configuration files. LibreChat uses three config files — .env for secrets, docker-compose.override.yml for container customisation, and librechat.yaml for app behaviour.

# Clone the repo
cd ~
git clone https://github.com/danny-avila/LibreChat.git
cd LibreChat

# Create config files from examples
cp .env.example .env
cp docker-compose.override.yml.example docker-compose.override.yml
cp librechat.example.yaml librechat.yaml

Edit the .env file with your settings:

nano .env
# Generate strong secrets (run these separately, paste the output)
# openssl rand -hex 32

JWT_SECRET=your-strong-random-secret
JWT_REFRESH_SECRET=your-strong-random-secret

# Registration — set true to create your first account,
# then change to false to lock down after setup
ALLOW_REGISTRATION=true
ALLOW_SOCIAL_LOGIN=false
Important: Never use weak or default JWT secrets in production. Run openssl rand -hex 32 twice to generate two unique secrets and paste them in.

Start all containers:

docker compose up -d

Verify all 5 containers are running:

docker compose ps
ContainerPurpose
LibreChatMain application (port 3080)
chat-mongodbChat history & user database
chat-meilisearchFull-text search across conversations
rag_apiDocument Q&A (RAG)
vectordbVector embeddings (pgvector)
Tip: At this point LibreChat is accessible at http://your-server-ip:3080. Register your admin account now before proceeding — then you'll lock down registration in a later step.
04

Set Up Subdomain DNS

DNS provider → add A record → point to Elastic IP

Before setting up SSL, you need a domain pointing to your server. I added an A record in my DNS provider (Namecheap/Cloudflare) for the subdomain chat.vigneshiam.com.

TypeHost / NameValueProxy
AchatYour Elastic IPDNS Only ☁️
If using Cloudflare: Keep this record set to DNS Only (grey cloud) — not proxied. Cloudflare's proxy can interfere with Let's Encrypt certificate verification and WebSocket connections that LibreChat uses.

Verify DNS has propagated before moving to the next step:

nslookup chat.yourdomain.com
# Should return your Elastic IP
05

Configure Nginx Reverse Proxy

install nginx → create site config → enable

LibreChat runs internally on port 3080. Nginx sits in front of it, accepting traffic on port 80/443 and forwarding it to LibreChat. It also handles WebSocket connections which LibreChat uses for real-time streaming.

sudo apt install nginx -y
sudo nano /etc/nginx/sites-available/librechat

Paste this configuration (replace with your subdomain):

server {
    listen 80;
    server_name chat.yourdomain.com;

    location / {
        proxy_pass         http://localhost:3080;
        proxy_http_version 1.1;
        proxy_set_header   Upgrade $http_upgrade;
        proxy_set_header   Connection 'upgrade';
        proxy_set_header   Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
# Enable the site
sudo ln -s /etc/nginx/sites-available/librechat \
           /etc/nginx/sites-enabled/

# Test config and restart
sudo nginx -t
sudo systemctl restart nginx
sudo systemctl enable nginx
06

Issue SSL Certificate

certbot → let's encrypt → auto HTTPS

Use Certbot to get a free SSL certificate from Let's Encrypt. It automatically updates the Nginx config to redirect HTTP to HTTPS and handles renewals every 90 days.

# Install Certbot with Nginx plugin
sudo apt install certbot python3-certbot-nginx -y

# Issue certificate (replace with your subdomain)
sudo certbot --nginx -d chat.yourdomain.com

Certbot will ask for your email (for renewal notices) and whether to redirect HTTP to HTTPS — choose yes to redirect. After completion your site is live at https://chat.yourdomain.com.

Auto-renewal: Certbot installs a cron job automatically. Certificates renew silently every 90 days. You can test it with:
sudo certbot renew --dry-run
07

Connect Google Gemini (Free Tier)

aistudio.google.com → API key → .env → librechat.yaml

LibreChat supports many AI providers. I used Google Gemini because it has a generous free tier. Get your API key from aistudio.google.com — no billing required for free tier models.

Add the key to .env:

nano ~/LibreChat/.env
GOOGLE_KEY=your-gemini-api-key-here

Restrict to free tier models only in librechat.yaml:

nano ~/LibreChat/librechat.yaml
endpoints:
  google:
    models:
      default: ["gemini-2.5-flash", "gemini-2.0-flash", "gemini-1.5-flash"]
      fetch: false
    modelDisplayLabel: "Google Gemini"

cd ~/LibreChat && docker compose restart
08

Custom Branding & Lock Down

librechat.yaml → branding → disable registration

Personalise the login page and lock down open registration so only you can use the instance.

Add to librechat.yaml:

interface:
  appTitle: "Your App Name"
  loginTitle: "Welcome to Your App"
  loginSubtitle: "Your personal AI assistant"
  footerText: "© 2026 yourname"
  socialLogins: []
  logo: "https://yourdomain.com/images/logo.png"

Once your admin account is created, disable open registration in .env:

ALLOW_REGISTRATION=false
# Restart to apply
cd ~/LibreChat && docker compose restart
Tip: With registration disabled, new users can only be created manually via the admin panel or MongoDB. This keeps your instance private.
Maintenance Reference

Useful commands to keep your LibreChat running smoothly.

Upgrade LibreChat

cd ~/LibreChat
docker compose down
git pull
docker compose pull
docker compose up -d

Quick MongoDB Backup

docker exec librechat-mongodb-1 \
  mongodump --out /tmp/backup-$(date +%Y%m%d)

Check Logs

cd ~/LibreChat
docker compose logs -f librechat
