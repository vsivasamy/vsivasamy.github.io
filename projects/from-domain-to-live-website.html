01

Purchase Your Domain

namecheap.com → search → buy

The first step is owning your domain name — your address on the internet, like yourname.com. I chose Namecheap for its clean interface, competitive pricing, and free WhoisGuard privacy protection.

What is a domain registrar?
A registrar is a company authorised to sell domain names. You pay an annual fee (typically $8–$15/year for a .com) to reserve it. You renew it each year.
  • Go to namecheap.com and search for your desired domain name
  • Add to cart — check WhoisGuard (privacy protection) is included free
  • Complete checkout — domain is active within minutes
  • You'll see it listed in your Namecheap dashboard under "Domain List"
Tip: Avoid domains with hyphens or numbers. Stick to .com if available — it's the most trusted extension globally.
02

Create a GitHub Repository

github.com → new repo → special naming

GitHub Pages is a free hosting service built into GitHub. For a personal site, you need a repository with a specific name — your GitHub username followed by .github.io.

Why this specific name?
GitHub automatically recognises a repo named username.github.io as your personal Pages site and serves it at that URL.

My GitHub username is vsivasamy, so my repo is:

vsivasamy/vsivasamy.github.io
  • Go to github.com → click "New repository"
  • Name it exactly: yourusername.github.io
  • Set it to Public (required for GitHub Pages on free plan)
  • Initialise with a README — then clone it locally
03

Build Your HTML Website

index.html → images/ → portfolio/

GitHub Pages serves static files — HTML, CSS, JavaScript, and images. No server-side code, no database. The entry point is always index.html at the root of your repo.

My repository structure:

vsivasamy.github.io/
├── index.html    ← main landing page
├── CNAME       ← custom domain pointer
├── images/
│   └── profile.png  ← profile picture
└── portfolio/
     └── index.html  ← My Story page
Tip: Each folder with an index.html becomes its own URL automatically. So portfolio/index.html is accessible at vigneshiam.com/portfolio/ — no extra config needed.
  • Create index.html at the root — this is your homepage
  • Add an images/ folder for assets like your profile photo
  • Add a portfolio/index.html for additional pages
  • Commit and push to GitHub — the site goes live within seconds
04

Enable GitHub Pages

repo settings → pages → deploy from branch

Once your files are pushed, tell GitHub to serve them as a website. This is a one-time setting.

  • Go to your repo on GitHub → click Settings
  • In the left sidebar, click Pages
  • Under "Source", select Deploy from a branch
  • Choose main branch and / (root) folder → click Save
  • After a minute, GitHub shows: "Your site is live at https://yourusername.github.io"
What happens under the hood?
GitHub runs a build pipeline every time you push. It serves your files from a global CDN with free HTTPS included automatically.
05

Move DNS to Cloudflare

cloudflare.com → add site → update nameservers

Before pointing your domain to GitHub Pages, I moved DNS management to Cloudflare — faster DNS, free CDN, DDoS protection, and a much cleaner dashboard.

  • Create a free account at cloudflare.com
  • Click "Add a Site" → enter your domain → choose the Free plan
  • Cloudflare scans and imports your existing DNS records automatically
  • Note the two Cloudflare nameservers provided
  • Go to Namecheap → Domain List → Manage → Nameservers → Custom DNS
  • Paste the Cloudflare nameservers and save — propagation takes a few hours
Tip: Keep your GitHub Pages A records set to DNS Only (grey cloud) in Cloudflare — not proxied. Proxying can interfere with GitHub's SSL certificate for your domain.
06

Point Your Domain to GitHub Pages

DNS records + CNAME file in repo

Two things to do: add DNS records in Cloudflare, and add a CNAME file in your repo.

Step 1 — Add DNS records in Cloudflare:

Type  Name  Value               Proxy
A     @     185.199.108.153     DNS Only
A     @     185.199.109.153     DNS Only
A     @     185.199.110.153     DNS Only
A     @     185.199.111.153     DNS Only
CNAME www   yourusername.github.io  DNS Only

Step 2 — Add a CNAME file in your repo:

Create a file named exactly CNAME (no extension) at the root of your repo with just your domain name:

vigneshiam.com

Step 3 — Set custom domain in GitHub Pages settings:

  • Repo → Settings → Pages → Custom domain → enter your domain → Save
  • Wait a few minutes for GitHub to verify DNS
  • Once verified, tick Enforce HTTPS — GitHub provisions a free SSL certificate
Result: Your site is now live at https://yourdomain.com — hosted free, secured with HTTPS, served globally. Any push to your main branch deploys automatically.
What You End Up With

  • A fast, globally hosted website — completely free
  • Automatic HTTPS with a GitHub-managed SSL certificate
  • Git-based deployments — push to deploy, no FTP, no servers
  • DNS managed via Cloudflare with DDoS protection built in
  • Expandable — add pages, a blog, or a portfolio anytime
